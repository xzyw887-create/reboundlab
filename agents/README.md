# Отдел агентов — ReboundLab (внутренний)

> Не для пользователей сайта. Общая память для всех «менеджеров» в Cursor.

## Как это работает

1. Каждый агент — **папка** с `BRIEF.md`, `PLAN.md`, `TASKS.md`, `STATUS.md`.
2. Перед работой агент читает свой `BRIEF` + `STATUS` (что уже придумано / в процессе).
3. После сессии агент обновляет `STATUS.md` и чеклист в `TASKS.md`.
4. UI «офиса» (MODULE_28) позже покажет эти же файлы — **источник правды уже здесь**.

## Агенты

| Агент | Папка | Зона ответственности |
|-------|-------|----------------------|
| Маркетинг | `agents/marketing/` | Продвижение, контент, waitlist, реклама |
| Продукт | `agents/product/` | Тарифы, UX, roadmap фич |
| Разработка | *(этот чат / код)* | Бэктестер, сайт, API |
| Финансы | `agents/finance/` | ИП/самозанятый, ЮKassa, цены |

## Промпт для входа в агента

```
Ты — [роль] ReboundLab. Прочитай agents/[имя]/BRIEF.md и STATUS.md.
Не дублируй уже записанное. Обнови STATUS после работы.
```
