# Решения по загрузке данных и тарифам

**Зафиксировано:** 2026-07-18 (ответы пользователя)

| # | Вопрос | Решение |
|---|--------|---------|
| 1 | Пробный период — 3 месяца | **Якорь = дата регистрации.** 90 дней назад от неё до сегодня. |
| 2 | Basic vs Pro — глубина | **Одинаково:** до 3 лет (или с листинга монеты). |
| 3 | Гость без регистрации | ✅ **Подтверждено:** демо как сейчас (Trial = 90 д, Basic/Pro preview = 3 года). |
| 4 | Частичная загрузка | **Грузим полностью** все пары × 1095 дней. |
| 5 | Таймфреймы в БД | ✅ **Подтверждено:** пока **только 1m** в БД. |
| 6 | Молодые монеты (<90 д) | **Нет** — не включаем. |

---

## П.3 — гость без регистрации

Человек **ещё не зарегистрировался**, но открыл сайт и в демо переключает тариф.

**Сейчас:**
- **Trial** в демо → **90 дней** от сегодня назад;
- **Basic / Pro** в демо → **до 3 лет** в календаре (превью тарифа).

**Решение:** оставляем так.

---

## П.5 — таймфреймы

**Сейчас:** в базе только **1m**. 5m / 15m / 1h собираются из минуток в UI и бэктесте.

**Если хранить 5m/15m/1h отдельно:** быстрее графики, но база в разы больше и backfill намного дольше.

**Решение:** остаёмся на **только 1m**.
