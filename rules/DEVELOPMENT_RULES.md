# DEVELOPMENT RULES — ReboundLab

## Этапы

1. **Проектирование** — только документы (✅ завершён)
2. **Разработка** — один модуль за раз
3. **Сборка** — интеграция готовых модулей

## Перед каждой задачей

1. Прочитать `docs/PROJECT_INDEX.md`
2. Прочитать только нужный `modules/MODULE_XX.md`
3. Перечислить файлы, которые будут открыты и изменены
4. Объяснить, зачем каждый файл

## Во время работы

- Работать ТОЛЬКО с указанным модулем
- Если можно решить одним файлом — не открывать второй
- Не рефакторить без разрешения
- Не добавлять зависимости без согласования
- Если задача выходит за модуль — остановиться и спросить

## После задачи

1. Обновить `docs/PROJECT_INDEX.md` (статус модуля)
2. Обновить `docs/ROADMAP.md` (чеклист)
3. Вывести: что изменено, почему, что осталось

## Контекст

- Никогда не анализировать весь проект
- Максимум: PROJECT_INDEX + один MODULE + файлы задачи
